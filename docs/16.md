## Estudos iniciais no Unsloth (Orpheus)

No primeiro eixo de estudo com Unsloth, estamos usando o modelo Orpheus ([ArtooDtoo/Orpheus_PTBR_FT_Unsloth_Chk-1800](https://huggingface.co/ArtooDtoo/Orpheus_PTBR_FT_Unsloth_Chk-1800)); uma das hipoteses para a degradacao observada e que o proprio checkpoint de base talvez ja nao ofereca qualidade suficiente como ponto de partida para este ajuste fino.

No Orpheus TTS, o fine-tuning com Unsloth nao exige um unico locutor e funciona com multiplos falantes, sendo o fator critico a consistencia do dataset, conforme a pratica observada no repositorio oficial ([canopyai/Orpheus-TTS](https://github.com/canopyai/Orpheus-TTS)) e em checkpoints publicos como [ArtooDtoo/Orpheus_PTBR_FT_Unsloth_Chk-1800](https://huggingface.co/ArtooDtoo/Orpheus_PTBR_FT_Unsloth_Chk-1800).
